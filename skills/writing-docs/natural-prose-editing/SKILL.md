# Natural Prose Editing

## Overview

Stiff, generic prose has identifiable mechanics — stock phrases, triad addiction, lockstep sentence rhythm, vague abstraction — and each has a mechanical fix. This is a writing-quality skill: the goal is prose worth reading, never disguise.

## When to Use

- The text is technically correct but reads flat, generic, or interchangeable with anything else on the topic.
- Someone says a draft "sounds like AI" or "doesn't sound like me."
- A blog post, email, or report needs a pass before it goes out.

**When NOT to use:**
- Evading AI-detection tools on academic or dishonest submissions — this skill is about writing quality, not detector evasion, and should be refused if that's the stated goal.

## Prerequisites

- The original draft or source text.
- Ideally, a few samples of the person's own writing (email, past posts) to match voice — without it, aim for plain, specific, unhedged prose rather than a fabricated "personality."

## The Workflow

1. **Read once for content, not style.** Confirm the claims and structure are sound before touching wording — don't polish something that's wrong.

2. **Strip cliché AI phrasing.** Flag and rewrite stock openers/closers and hedges:
   - "It's important to note that…" → just state the thing.
   - "In today's fast-paced world…" → cut entirely, start with the point.
   - "This isn't just X, it's Y" (false-contrast construction) → say what it actually is.
   - "Overall," "In conclusion," "Furthermore" as paragraph openers → delete or replace with a real transition tied to content.
   - Excessive hedging ("could potentially," "it's worth considering") → commit to a claim or cut it. Exception: keep the hedge where genuine uncertainty exists — sometimes "might" is the accurate word.

3. **Break up rule-of-three / triad addiction.** AI-generated text leans hard on three-item lists and three-clause sentences ("clear, concise, and compelling"). Vary it: some sentences get one strong word, some get a longer list, some get none.

4. **Vary sentence length and rhythm.** Look for runs of same-length, same-structure sentences. Break some short. Let one run long. Read it out loud — natural writing doesn't march in lockstep.

5. **Replace vague abstractions with concrete specifics.** "This leads to better outcomes" → name the outcome. "Many users report issues" → say which issue, roughly how many, or cut the unsupported claim.

6. **Cut throat-clearing.** Delete sentences that restate the topic before getting to the point ("When it comes to X, there are many factors to consider"). Start with the actual first fact or claim.

7. **Match voice, if a sample exists.** Check contractions, sentence length, formality, and characteristic phrases against the person's real writing samples; adjust rather than imposing a generic "conversational" tone. No sample → plain and specific, never a fabricated personality.

8. **Read the final pass end to end.** Confirm nothing was smoothed into blandness — natural prose still has an edge or opinion, not just fewer clichés.

## Common Rationalizations

| Excuse | Why it doesn't hold |
|---|---|
| "Just swap out the banned words — same effect, faster" | The underlying problem is structure and rhythm, not any single phrase. Find-and-replace produces the same flat prose with different furniture. |
| "Add some slang and exclamation points — instantly human" | Forced quirkiness reads worse than the original stiffness. Natural means specific and committed, not costumed. |
| "Remove every hedge — confidence sounds human" | Where genuine uncertainty exists, "might" is the accurate word, and deleting it makes the text wrong, not natural. Cut performative hedges; keep honest ones. |
| "No voice sample? I'll give it a friendly personality" | A fabricated 'conversational' tone isn't the person's voice — it's a different generic. Without a sample, the target is plain, specific, unhedged prose. |
| "The client just needs it to pass a detector" | Out of scope, decline. This skill improves writing; helping disguise authorship for academic or dishonest submissions is a different request wearing this one's vocabulary. |
| "Content's fine, it just needs style polish" | Polishing wrong claims produces confident wrong claims. The content-first read (step 1) is why the edit starts before the wording does. |

## Red Flags

- The edit is a word-substitution table applied mechanically.
- Output tone wildly unlike the person's actual samples (or a "personality" invented without samples).
- Every hedge removed, including ones marking real uncertainty.
- The request's context is an academic submission or authorship disguise.
- Edited text has fewer clichés but the same lockstep rhythm (structure untouched).
- All edge and opinion smoothed out — blander than the input.

## Verification

- [ ] Content pass done first — factual/structural issues flagged before style edits.
- [ ] Stock-phrase sweep complete — before/after examples visible in the diff.
- [ ] Rhythm varied — no runs of 3+ same-structure sentences in the result (read-aloud check done).
- [ ] Vague claims made concrete or cut — each "better/many/issues" resolved to specifics.
- [ ] Voice matched against provided samples (or plain-prose default stated) — noted.
- [ ] End-to-end read confirms edge/opinion preserved.

## Example

**Before:**
> It's important to note that in today's competitive landscape, effective
> communication is not just important, it's essential. Furthermore, clear,
> concise, and compelling writing can help you stand out.

**After:**
> Clear writing gets read. Everything else is competing for an inbox that's
> already full — so cut the sentence that doesn't earn its place.
